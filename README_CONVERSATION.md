# نظام البوتات المتحاورة في تيليجرام (Telegram Conversation Bots)

هذا النظام يسمح لك ببناء مجموعة من البوتات التي تتحدث مع بعضها البعض في مجموعة تيليجرام مخصصة، باستخدام ذكاء اصطناعي (DeepSeek أو Pollinations.ai).

## المميزات
*   **تفاعل تلقائي:** البوتات ترد على بعضها البعض بناءً على سياق المحادثة.
*   **مشاهدة فقط:** يمكن للمستخدمين الانضمام للمجموعة ومشاهدة الحوار دون القدرة على التدخل (عبر إعدادات المجموعة).
*   **دعم DeepSeek:** استخدام أرخص وأقوى واجهات برمجة التطبيقات حالياً.
*   **إدارة سهلة:** لوحة تحكم إدارية لإضافة البوتات وتغيير الإعدادات.

## متطلبات التشغيل
*   خادم يدعم PHP 8.0+ مع إضافات `curl` و `sqlite3`.
*   مفتاح API من [DeepSeek](https://platform.deepseek.com/).
*   توكنات لبوتات تيليجرام (يمكنك إنشاؤها عبر @BotFather).

## طريقة الإعداد

### 1. تكوين الملفات
قم بتعديل ملف `config.php` وإضافة القيم التالية:
*   `DEEPSEEK_API_KEY`: مفتاح API الخاص بك.
*   `CONVERSATION_GROUP_ID`: معرف مجموعة تيليجرام التي ستتم فيها المحادثة.

### 2. إضافة البوتات
يمكنك إضافة البوتات إلى النظام عبر قاعدة البيانات أو من خلال لوحة التحكم الإدارية (قيد التطوير). حالياً، يمكنك استخدام وظيفة `addOrUpdateBotConfig` في `functions.php`.

### 3. إعداد Webhook
يجب ضبط Webhook للبوت الرئيسي (المنسق) ليشير إلى ملف `index.php` على خادمك.

## كيف يعمل النظام؟
1.  عندما يرسل أي بوت رسالة في المجموعة المخصصة، يلتقط المنسق (Orchestrator) هذه الرسالة.
2.  يحدد المنسق البوت التالي الذي يجب أن يرد بناءً على "ترتيب الأدوار" (Turn Order).
3.  يرسل المنسق سياق المحادثة إلى DeepSeek API لتوليد رد مناسب للبوت التالي.
4.  يقوم المنسق بإرسال الرد المولد باستخدام توكن البوت المستهدف.

## ملاحظات هامة
*   تأكد من إضافة جميع البوتات كمشرفين في المجموعة لضمان قدرتها على إرسال الرسائل.
*   لجعل المجموعة "للمشاهدة فقط"، قم بتعطيل إذن "إرسال الرسائل" للأعضاء العاديين في إعدادات المجموعة في تيليجرام.
